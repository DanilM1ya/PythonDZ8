# Домашнее задание: Анализ трафика и XSS
Студент: Данил Мухутдинов

## Выполнено в WSL2 (Ubuntu)

### Этап 1: Настройка окружения
- Использован предустановленный WSL2 в Windows
- Установлены: tcpdump, iptables, Scapy
- Перехват выполнялся на интерфейсе loopback (lo)

### Этап 2: Анализ трафика
- Запущен локальный HTTP-сервер на порту 8000
- Перехвачен реальный сетевой трафик с помощью tcpdump
- Сохранён файл: xss_traffic.pcap

### Этап 3: Эксплуатация XSS
Выполнены реальные HTTP-запросы с полезными нагрузками:
  curl "http://localhost:8000/?q=%3Cscript%3Ealert(1)%3C/script%3E"
  curl "http://localhost:8000/?q=%3Cimg%20src=x%20onerror=alert(2)%3E"

(%3C = '<', %3E = '>' — URL-кодирование спецсимволов)

### Этап 4: Анализ результатов
В перехваченном трафике обнаружены признаки XSS:
- URL-кодированные теги %3Cscript%3E в параметре q
- Обработчик события onerror= в URL-кодированном виде
- Payload передаётся в открытом виде в GET-параметре

Изменения в трафике при атаке:
1. В строке запроса появляются спецсимволы в URL-кодированном виде (%3C, %3E)
2. Длина URL увеличивается за счёт вставки тегов и обработчиков
3. В теле запроса присутствует неэкранированный пользовательский ввод
